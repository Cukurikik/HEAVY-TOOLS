## 2024-05-24 - Semantic Buttons & Focus Rings in Navigation
**Learning:** Pure visual indicators are insufficient for accessibility. Adding `focus-visible` ensures keyboard navigation users can orient themselves without impacting mouse users. Additionally, converting interactive `div` elements (like command search bars) into semantic `<button>` elements naturally handles keyboard focus and space/enter key interactions, eliminating the need to manually implement these behaviors.
**Action:** Always prefer semantic HTML tags (`<button>`, `<a>`) for interactive elements, and pair them with `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-indigo-500` (or appropriate accent color) to provide robust keyboard accessibility out-of-the-box. Ensure all icon-only buttons include descriptive `aria-label`s.
