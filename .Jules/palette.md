## 2024-05-18 - Added proper button roles to Interactive Divs
**Learning:** Custom UI elements acting as buttons (like ToolCardComponent and FileDropZoneComponent) need explicit `role="button"`, `tabindex="0"`, `focus-visible` utility classes, and `(keydown.space)` event handling alongside `(keydown.enter)` and `(click)` to provide equivalent accessibility to native buttons. Additionally, the `@angular-eslint/template/no-negated-async` rule strictly forbids `!(obs$ | async)` requiring explicit boolean checks or `@if(obs$ | async; as ...)` blocks.
**Action:** When creating or fixing custom interactive elements in this app's design system, immediately include `role="button"`, `focus-visible:ring-2` (using the primary accent color like `focus-visible:ring-accent-cyan`), and `(keydown.space)` to ensure full keyboard navigation support without violating strict linting rules.
